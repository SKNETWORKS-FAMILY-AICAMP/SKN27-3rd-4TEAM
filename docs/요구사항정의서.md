# 전세계약 위험 진단 서비스 요구사항 정의서

## 1. 문서 개요

본 문서는 전세계약 위험 진단 및 전세사기 예방 서비스의 기능 요구사항, 비기능 요구사항, 화면 요구사항, 데이터 요구사항을 정의한다. 서비스는 사용자가 임대차계약서 또는 계약 정보를 입력하면 실거래가 데이터, 계약서 특약, 법령·판례 근거를 종합하여 전세 계약의 위험도를 진단하고 대응 방안을 제공하는 것을 목표로 한다.

---

## 2. 서비스 개요

### 2.1 서비스명

안전계약 전세사기 진단 서비스

### 2.2 목적

- 임차인이 전세계약 전 위험 요소를 사전에 확인할 수 있도록 지원한다.
- 계약서와 입력 정보를 기반으로 전세가율, 지역 시세 대비 보증금, 특약 위험, 법률 쟁점을 종합 진단한다.
- 진단 결과를 바탕으로 안전 체크리스트, 대응 절차, AI 상담을 제공한다.

### 2.3 주요 사용자

| 사용자 | 설명 |
|---|---|
| 임차인 | 전세계약 전 또는 계약 진행 중 위험 여부를 확인하려는 사용자 |
| 예비 임차인 | 매물 계약 전 시세·권리·특약 위험을 검토하려는 사용자 |
| 상담자/운영자 | 사용자의 진단 결과와 상담 이력을 참고하여 안내하는 담당자 |

---

## 3. 서비스 범위

### 3.1 포함 범위

- 임대차계약서 PDF/DOCX 업로드
- 주소, 전세금, 면적 직접 입력
- 계약서 핵심 정보 추출
- 종로구 실거래가 기반 가격 위험 분석
- 전세가율 및 지역 평균 대비 보증금 분석
- 특약 조항 위험 분석
- 판례·법조문 기반 RAG 상담
- 진단 리포트 생성 및 저장
- 진단 기록 조회
- 체크리스트 제공
- 피해 대응 플레이북 제공
- 지역별 시세 화면 제공

### 3.2 제외 범위

- 실제 법률 자문 또는 변호사 대리 행위
- 보증보험 가입 신청 대행
- 등기부등본 실시간 발급 연동
- 실시간 국토부 API 자동 동기화
- 결제 및 사용자 인증

## 9. 제약사항 및 가정

- 본 서비스의 결과는 법률 자문이 아닌 참고 정보이다.
- 실거래 데이터 범위는 현재 프로젝트 기준 서울 종로구 중심이다.
- LLM 응답은 API 상태, 모델, 프롬프트에 따라 일부 변동될 수 있다.
- 판례 GraphDB와 VectorDB는 사전에 데이터 적재가 되어 있어야 충분한 근거 검색이 가능하다.
- 사용자가 업로드한 계약서의 품질이 낮거나 스캔 이미지 기반이면 텍스트 추출 정확도가 낮아질 수 있다.

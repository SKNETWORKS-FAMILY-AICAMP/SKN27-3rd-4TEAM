# RAGAS 기반 RAG 평가 결과

## 1. 평가 개요

- 평가 대상: 전세계약 위험 진단 RAG 파이프라인
- 평가 방식: FastAPI 서버 호출 없이 로컬 RAG 함수(`generate_answer`)를 직접 호출
- 평가 도구: RAGAS `0.4.3`
- 평가 모델: `gpt-4o-mini`
- 임베딩 모델: `text-embedding-3-small`
- 검색 문서 수: 질문당 `3개`
- 평가 케이스 수: `4개`

## 2. 평가 실행 방식

```powershell
.\.venv\Scripts\python.exe rag\ragas_test.py
```

`ragas_test.py`는 다음 흐름으로 평가를 수행한다.

1. 평가 질문을 로컬 RAG 파이프라인에 전달한다.
2. RAG 검색 결과와 생성 답변을 수집한다.
3. `question`, `answer`, `contexts`, `ground_truth` 형식의 RAGAS 평가 데이터셋을 구성한다.
4. RAGAS 공식 지표로 답변 품질과 검색 문맥 품질을 평가한다.

## 3. 평가 지표 결과

| 지표 | 점수 | 의미 |
|---|---:|---|
| Faithfulness | 0.8500 | 답변이 검색된 근거 문서에 얼마나 충실한지 |
| Answer Relevancy | 0.3044 | 답변이 질문 의도에 얼마나 직접적으로 부합하는지 |
| Context Precision | 1.0000 | 검색된 문맥 중 질문 해결에 유용한 문맥의 비율 |
| Context Recall | 1.0000 | 정답에 필요한 근거가 검색 문맥에 얼마나 포함되었는지 |

## 4. 결과 해석

검색 품질은 매우 높게 측정되었다. `Context Precision`과 `Context Recall`이 모두 `1.0000`으로 나와, 검색된 문서가 평가 질문의 정답 근거를 충분히 포함하고 있음을 확인했다.

답변의 근거 충실도도 `Faithfulness 0.8500`으로 비교적 양호하다. 이는 생성 답변이 검색된 문서 내용을 대체로 잘 기반으로 삼고 있다는 의미다.

다만 `Answer Relevancy`가 `0.3044`로 낮게 측정되었다. 이는 답변이 문서 근거에는 충실하지만, 질문에 대해 더 직접적이고 간결하게 답변하지 못했거나 질문 핵심과 일부 어긋난 문장이 포함되었을 가능성을 의미한다.

## 5. 종합 판단

- 검색 단계는 안정적으로 동작한다.
- 검색된 context는 정답 근거를 충분히 포함한다.
- 생성 답변은 근거 기반성은 양호하지만, 질문 의도에 맞춘 답변 직접성이 부족하다.
- 따라서 현재 RAG는 “검색 성능은 양호, 답변 생성 프롬프트 개선 필요”로 판단된다.

## 6. 개선 방향

1. 답변 프롬프트에 “질문에 대한 핵심 답변을 먼저 1~2문장으로 제시”하도록 명시한다.
2. 근거 문서 요약보다 질문에 직접 답하는 형식을 강화한다.
3. 답변에 불필요한 부연 설명이 많아지지 않도록 문장 수 또는 구조를 제한한다.
4. 평가 케이스를 4개에서 더 늘려 법률, 보증보험, 등기부, 특약 등 유형별 점수를 분리 측정한다.

## 7. 결론

현재 RAG 파이프라인은 검색 근거 확보 측면에서는 높은 성능을 보였으며, 답변도 근거 문서에 대체로 충실했다. 다만 질문과 답변의 직접적인 관련성을 높이기 위해 생성 프롬프트 개선이 필요하다.
